# 실전 사용법 — "내일 출근해서 뭘 하면 되나"

## 전체 흐름 (한눈에)

```
Day 1          Week 1~2         Week 3~4         Week 5~
─────────────────────────────────────────────────────────
업무 점수화  →  프롬프트 실전 적용  →  파이썬 자동화  →  팀 공유
(30분)         + KPI 기록 시작      파이프라인 연결     플레이북 배포
```

---

## Step 1. 업무 점수화 (내일 당장, 30분)

`01-task-audit/task-scoring-template.csv`를 엑셀이나 구글시트로 연다.

### 하는 법

1. 지금 들어있는 샘플 업무 9개는 **증권사 분석팀 일반 예시**. 본인 업무로 바꿔야 한다
2. 이번 주에 한 일을 떠올려서 한 줄씩 적는다
3. 반복성 / AI가능성 / 영향도를 각각 1~5점으로 솔직하게 매긴다
4. **우선순위점수 = 세 점수를 곱함** (최대 125점)
5. **60점 이상인 업무 중 상위 3개**를 첫 파일럿으로 픽한다

### 예시

| 업무 | 반복성 | AI가능성 | 영향도 | 점수 | 판정 |
|------|--------|----------|--------|------|------|
| 주간 수익률 리포트 | 5 | 5 | 4 | **100** | 1순위 |
| 이상거래 탐지 리뷰 | 5 | 4 | 5 | **100** | 1순위 |
| 경영진 보고자료 | 4 | 4 | 5 | **80** | 2순위 |
| 캠페인 성과 분석 | 3 | 3 | 4 | 36 | 후순위 |

점수가 높은 3개를 골라서 팀장에게 "이걸 AI로 개선하겠다"고 제안한다.

---

## Step 2. 프롬프트 템플릿 바로 써보기 (첫 주)

`04-team-playbook/prompt-templates/` 안의 5개 프롬프트가 가장 바로 쓸 수 있는 무기다. **코드 설치 없이 ChatGPT/Claude에 그대로 복붙하면 된다.**

### 프롬프트 5종 — 언제 쓰나

| 파일 | 언제 쓰나 | 체감 효과 |
|------|-----------|-----------|
| `01-eda.md` | 새 데이터를 처음 받았을 때 | 탐색 시간 50% 절감 |
| `02-report-summary.md` | 주간/월간 리포트 쓸 때 | 작성 시간 60% 절감 |
| `03-hypothesis-generation.md` | "왜 이런 현상이?" 분석할 때 | 가설 도출 시간 절감 |
| `04-anomaly-interpretation.md` | 이상치가 탐지됐을 때 | 1차 원인분석 자동화 |
| `05-executive-briefing.md` | 임원 보고자료 만들 때 | 보고 문구 초안 자동화 |

### 실전 예시 — 주간 리포트 요약문

`02-report-summary.md`의 프롬프트에서 `{중괄호}` 부분만 본인 데이터로 바꿔 넣는다:

```
당신은 증권사 데이터분석팀의 시니어 애널리스트입니다.
아래 데이터를 바탕으로 주간 리포트 요약문을 작성하세요.

## 보고 대상
- 수신자: 팀장
- 보고 목적: 정기보고

## 핵심 지표 데이터
| 지표 | 금주 | 전주 | 증감률 |
|------|------|------|--------|
| AUM | 152조원 | 148조원 | +2.7% |
| 순유입 | 1.2조원 | 0.8조원 | +50% |
| 활성고객 | 23만명 | 22.5만명 | +2.2% |

## 주요 이벤트
- 금리 인하 기대감으로 채권형 상품 유입 증가
- 신규 글로벌ETF 3종 출시

(이하 작성 규칙, 출력 형식은 템플릿 그대로 유지)
```

AI가 **"핵심 요약 → 주목 포인트 → 보수/중립/공격 권고안 3가지"** 구조로 초안을 뱉어준다.
거기서 숫자 틀린 거 잡고, 어투 다듬으면 끝.

### 프롬프트 사용 팁

- `{중괄호}` 안의 내용만 매번 바꾸고, 나머지 구조는 건드리지 않는다
- 데이터는 표(마크다운 테이블)로 넣어야 AI가 정확하게 읽는다
- "주요 이벤트"를 넣어주면 맥락 있는 해석이 나온다
- 결과가 마음에 안 들면 프롬프트를 고치지 말고 **데이터를 더 넣어본다**

## Step 3. KPI 추적하기 (매주 금요일, 10분)

`03-kpi-dashboard/weekly-kpi-tracker.csv`를 엑셀로 연다.

### 기록하는 법

**"AI적용전" 열**에는 기존 방식으로 걸린 시간을, **"AI적용후" 열**에는 AI 써서 걸린 시간을 적는다.

| 주차 | 지표 | AI적용전 | AI적용후 | 절감량 | 절감률 |
|------|------|----------|----------|--------|--------|
| W01 | 주간리포트작성(h) | 4.0 | 1.5 | 2.5 | 62% |
| W01 | 이상탐지대응(min) | 45 | 18 | 27 | 60% |
| W01 | 질의대응(min) | 30 | 12 | 18 | 60% |
| W02 | 주간리포트작성(h) | 4.0 | 1.3 | 2.7 | 67% |

### 대시보드 만들기 (4주치 쌓이면)

```bash
cd ai-analyst-strategy/03-kpi-dashboard
pip install pandas plotly
python kpi-visualizer.py --output dashboard.html
open dashboard.html
```

브라우저에서 다크 테마 대시보드가 열린다. 스코어카드 4개 + 차트 4종.

### 이걸 왜 하나

팀장에게 이렇게 말하기 위해서다:

> "AI 적용 4주 만에 리포트 작성 4시간 → 1.5시간(62% 절감), 누적 10시간 절감 = 전임 분석가 0.25명분입니다."

감이 아니라 **숫자로 증명**해야 인정받는다.

## Step 4. 파이썬 파이프라인 연결하기 (2주차~)

프롬프트로 효과가 확인된 업무부터 코드 자동화로 넘어간다.

### 환경 세팅

```bash
cd ai-analyst-strategy/02-quick-wins
pip install -r requirements.txt
```

### 3개 파이프라인 — 본인 환경에 맞추는 법

#### 1) `01_report_automation.py` — 리포트 자동화

수정할 곳 **딱 3군데**:

| 위치 | 뭘 바꾸나 | 예시 |
|------|-----------|------|
| `CONFIG["db_path"]` | DB 연결 경로 | `"postgresql://user:pw@host/db"` |
| `REPORT_QUERIES` | SQL 쿼리 | 본인 테이블/컬럼명으로 교체 |
| `CONFIG["llm_api_key"]` | LLM API 키 | 환경변수 `LLM_API_KEY` 설정 |

실행:
```bash
python 01_report_automation.py --report weekly --date 2026-03-04
```

결과: `reports/weekly_2026-03-09/` 폴더에 차트 HTML + 요약문 + 메타데이터 자동 생성.

#### 2) `02_anomaly_detection.py` — 이상탐지

수정할 곳:

| 위치 | 뭘 바꾸나 |
|------|-----------|
| `METRIC_DEFINITIONS` | 모니터링할 지표 SQL과 컬럼명 |
| `CONFIG["alert_webhook"]` | Slack/Teams 알림 웹훅 URL |

실행:
```bash
python 02_anomaly_detection.py --target trades --lookback 30 --date 2026-03-04
```

결과: 이상치 감지되면 `anomaly_reports/` 폴더에 마크다운 보고서 + CSV 생성.

#### 3) `03_query_assistant.py` — 질의 대응

수정할 곳:

| 위치 | 뭘 바꾸나 |
|------|-----------|
| `QUERY_PATTERNS` | 질문 유형별 SQL 템플릿 |
| `SEGMENT_MAP` | 회사 고객 세그먼트 기준 |

실행:
```bash
python 03_query_assistant.py "이번 달 VIP 고객 수익률 알려줘"
```

결과: 질문 분류 → SQL 생성 → 검증 → 실행 → 답변 초안 출력.

### 핵심 포인트

**코드를 처음부터 다 고칠 필요 없다.** SQL 쿼리 부분과 DB 연결 부분만 본인 환경으로 바꾸면 나머지 로직(시각화, 검증, 요약 생성)은 그대로 동작한다.

## Step 5. 팀에 공유하기 (4주차~)

혼자 써보고 성과가 검증되면 팀으로 확산한다.

### 공유 순서

1. `04-team-playbook/playbook.md`를 **팀 Confluence/Notion에 복붙**
2. `prompt-templates/` 5개를 **팀 공유 폴더에 올림**
3. 20분짜리 데모를 잡고 **"전후 비교"를 보여줌**

### 데모에서 보여줄 것

- "이전: 리포트 4시간 → 지금: 1.5시간 (62% 절감)"
- "이전: 이상치 원인분석 45분 → 지금: AI 초안 18분 (60% 절감)"
- "4주 누적 절감: 40시간 = 전임 분석가 1명분"
- KPI 대시보드 실제 화면 시연

### 데모 후 액션

- 팀원들에게 프롬프트 템플릿 먼저 써보라고 권유 (진입장벽 낮음)
- 쓰는 사람이 3명 넘으면 → 본인이 "AI 활용 리드" 포지션 확보
- 플레이북에 팀원 피드백 반영 → 지속적 개선 사이클

---

## 가장 중요한 한 가지

이 키트의 **진짜 핵심은 코드가 아니다.** 코드는 도구일 뿐이고, 핵심은 이것이다:

> **"나는 AI로 이만큼의 시간을 절감했고, 이만큼의 비즈니스 임팩트를 만들었다"를 숫자로 말할 수 있느냐.**

그래서 순서가 중요하다:

| 순서 | 할 일 | 왜 이 순서인가 |
|------|-------|----------------|
| 1 | **프롬프트 템플릿** 당장 써라 | 설치 불필요, 오늘 가능 |
| 2 | **KPI 시트** 기록을 시작해라 | 숫자가 쌓여야 증거가 됨 |
| 3 | **파이썬 파이프라인** 연결해라 | 효과 검증된 업무부터 자동화 |
| 4 | **플레이북**으로 팀에 공유해라 | 개인기 → 팀 자산 전환 |

**프롬프트 템플릿 하나 복사해서 오늘 리포트 쓸 때 써보는 것 — 그게 Day 1의 전부다.**
