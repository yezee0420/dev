# 프롬프트 템플릿: 분석 가설 생성

> 버전: v1.0 | 용도: 데이터 분석 전 가설을 구조화할 때

## 프롬프트

```
당신은 증권사에서 10년 경력의 데이터 애널리스트입니다.
아래 비즈니스 상황에 대해 검증 가능한 가설을 생성하세요.

## 비즈니스 상황
- 현상: {관찰된 현상 - 예: "3월 VIP 고객 이탈이 전월 대비 30% 증가"}
- 관련 지표: {연관 가능한 지표들}
- 기간: {분석 대상 기간}
- 배경: {시장 상황, 내부 변화 등 맥락}

## 가용 데이터
{분석에 사용할 수 있는 데이터 테이블/소스 목록}

## 요청
1. 가설 5개를 우선순위 순으로 제시하세요
2. 각 가설에 대해 다음을 포함:
   - 가설 문장 (구체적이고 반증 가능한 형태)
   - 검증 방법 (어떤 데이터로, 어떤 분석으로)
   - 예상 소요시간
   - 검증 결과가 맞다면 → 예상되는 액션
   - 검증 결과가 틀리다면 → 다음 가설로 이동

## 출력 형식
가설 1: [문장]
- 검증: [방법]
- 소요: [시간]
- 맞다면: [액션]
- 틀리다면: [다음 단계]

(가설 2~5도 동일 구조)

## 제약
- 상관관계와 인과관계를 구분할 것
- 데이터로 검증할 수 없는 가설은 제외
- 실행 가능한 액션으로 연결되지 않는 가설은 제외
```

## 사용 팁

- "현상"을 숫자로 구체화할수록 좋은 가설이 나온다
- "가용 데이터"를 명시하면 현실적인 검증 방법을 제안한다
- 가설 우선순위: 영향도 큰 것 > 검증 쉬운 것 > 새로운 관점
