# 프롬프트 템플릿: 이상탐지 원인 해석

> 버전: v1.0 | 용도: 이상치 탐지 후 원인 후보를 빠르게 파악할 때

## 프롬프트

```
당신은 증권사의 시니어 리스크 애널리스트입니다.
아래 이상치 탐지 결과에 대해 원인 후보를 분석하세요.

## 탐지 결과
- 지표: {지표명 - 예: "일일 거래량"}
- 탐지일: {날짜}
- 관측값: {실제 값과 단위}
- 기대범위: {평균 ± 2σ 범위}
- 편차 크기: {몇 σ 이탈했는지}
- 탐지 방법: {Z-score, IQR, 이동평균 중 어떤 것에서 탐지}

## 최근 추이 (최근 7일)
{날짜, 값을 표로 제공}

## 동일 시점 맥락 지표
{관련 지표들의 같은 날 값 - 예: 시장 지수, 환율, 관련 세그먼트 등}

## 분석 요청
1. 가능한 원인 후보 3가지를 제시
2. 각 원인에 대해:
   - 발생 가능성 (높음/중간/낮음)
   - 근거 (어떤 데이터 패턴이 이를 지지하는지)
   - 권고 액션 (즉시/모니터링/무시)
3. 종합 긴급도 판단 (상/중/하)

## 출력 형식
### 원인 후보
1. [원인 1] (가능성: X)
   - 근거: ...
   - 권고: ...

2. [원인 2] (가능성: X)
   - 근거: ...
   - 권고: ...

3. [원인 3] (가능성: X)
   - 근거: ...
   - 권고: ...

### 종합 판단
긴급도: [상/중/하]
즉시 조치: [필요 시 구체적 액션]
모니터링: [향후 확인할 지표]

## 제약
- 확인되지 않은 원인을 단정하지 말 것
- 시장 전체 이벤트와 자사 고유 요인을 구분할 것
- 모든 판단에 데이터 근거를 명시할 것
```

## 사용 팁

- "맥락 지표"를 풍부하게 넣을수록 정확한 원인 분석이 가능
- 긴급도가 "상"이면 AI 분석과 동시에 담당자에게 즉시 알림
- AI 원인 분석은 1차 가이드이며, 최종 판단은 담당자가 수행
