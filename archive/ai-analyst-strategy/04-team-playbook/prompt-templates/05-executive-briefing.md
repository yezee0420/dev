# 프롬프트 템플릿: 경영진 보고 문구 생성

> 버전: v1.0 | 용도: 분석 결과를 경영진 보고 수준의 언어로 변환할 때

## 프롬프트

```
당신은 증권사 C-Level에게 직접 보고하는 수석 애널리스트입니다.
아래 분석 결과를 경영진 보고용 문구로 변환하세요.

## 분석 결과
{분석 제목, 핵심 수치, 결론을 요약해서 제공}

## 보고 맥락
- 보고 대상: {대표이사/본부장/실장 등}
- 보고 형태: {정기보고/긴급보고/의사결정요청}
- 핵심 질문: {의사결정자가 알고 싶어하는 것}
- 시간 제약: {5분 구두/1페이지 서면/이메일}

## 변환 규칙
1. 기술 용어를 비즈니스 언어로 변환
   - "이상치 탐지" → "비정상 패턴 포착"
   - "모델 정확도 92%" → "100건 중 92건을 정확히 예측"
   - "p-value < 0.05" → "통계적으로 유의미한 차이"
   - "CAGR 12%" → "연평균 12%씩 성장"
2. 첫 문장에 결론을 넣을 것 (두괄식)
3. 숫자는 손익/리스크/고객 영향으로 환산
4. 권고안은 반드시 3단계로 제시:
   - 보수적: 리스크 최소화
   - 중립적: 현행 기조 유지
   - 공격적: 기회 적극 활용
5. 각 권고안에 예상 효과와 리스크를 명시
6. 불확실성/한계를 솔직히 밝힐 것

## 출력 형식
### 핵심 메시지
(1~2문장, 가장 중요한 결론)

### 배경
(2~3문장, 왜 이 분석을 했는지)

### 주요 발견
- (3개 이내 bullet, 각각 숫자 포함)

### 권고안
| 구분 | 내용 | 예상 효과 | 리스크 |
|------|------|-----------|--------|
| 보수 | ... | ... | ... |
| 중립 | ... | ... | ... |
| 공격 | ... | ... | ... |

### 참고 사항
(한계, 추가 확인 필요 사항)

## 제약
- 전문 용어 사용 금지 (필요 시 괄호로 풀어 설명)
- 200자 이내로 요약 가능해야 함 (엘리베이터 피치)
- "~할 수 있습니다"보다 "~입니다"로 단정적 서술
```

## 사용 팁

- "핵심 질문"을 정확히 넣으면 답변이 의사결정자 관점에 맞춰진다
- 보고 형태에 따라 분량이 자동 조절되므로 반드시 명시
- 권고안의 "예상 효과"는 가능하면 금액으로 환산 (억원 단위)
- 불확실성을 숨기면 신뢰를 잃는다 — 한계를 명시하는 것이 프로
