# 게임 개발 고도화 체크리스트

게임을 만들거나 개선할 때 아래 요구 사항들을 짚어보면 더 완성도 높은 결과를 낼 수 있습니다.

---

## 1. 플레이 가능성 (난이도·공정성)

- **최소 1개 조합 보장**: 플레이어가 항상 한 번 이상 성공할 수 있도록, 초기 보드·리필 시 유효한 조합이 반드시 존재하게 할 것.
- **리프레시/힌트**: 막혔을 때 사용할 수단(리프레시, 스크램블, 힌트 등)을 정하고, 사용 횟수·조건을 명확히 할 것.
- **패배 조건**: 리프레시를 다 썼는데도 조합이 없을 때 등, 게임 오버 조건을 명확히 할 것.

**예시 요청**: "적당히 10을 만들 수 있게 잘 만들어놔야지. 지금은 10을 만들 수가 아예 없잖아", "최소 한 개의 조합은 항상 존재해야 돼"

---

## 2. 조작·입력 UX

- **드래그 자국 제거**: `user-select: none`, `user-drag: none`, 터치/마우스 이벤트에서 `preventDefault`로 선택·드래그 흔적이 안 남게 할 것.
- **선택 방식 명확화**: 직사각형 선택 vs 스네이크(인접 경로) 선택 등, 의도한 선택 방식이 구현되었는지 확인할 것.

**예시 요청**: "뒤에 드래그한 자국이 남는 게 그거 없어지게 해줘. 게임에 집중이 안 돼", "상하좌우로 스네이크처럼 이동이 가능해야 돼"

---

## 3. 시각 디자인

- **요소 단순화**: 직사각형·불필요한 테두리 제거, 아이콘만으로 의미가 전달되게 할 것.
- **숫자·정보 가독성**: 숫자·텍스트가 아이콘 위에서 잘 보이게(색 대비, 크기, 위치) 조정할 것.
- **선택 피드백**: 선택된 항목이 뚜렷하게 보이게(테두리, 확대, 색, 그림자 등) 강조할 것.

**예시 요청**: "사과 안에 숫자가 들어가게 수정해줘", "아주 단순한 사과 모양 아이콘이 있고 직사각형은 필요 없음. 숫자 잘 보이게", "선택한 사과가 좀 더 잘 보였으면 좋겠어"

---

## 4. 피드백·보상

- **플로팅 점수**: 제거·획득 시 점수를 해당 위치에서 잠깐 떠오르게 표시할 것.
- **표시 위치**: 마지막으로 제거된(또는 터치한) 위치 기준으로 표시할 것.
- **가독성**: 사과·다른 요소에 가려지지 않게 z-index·크기·배경(박스)으로 강조할 것.
- **점수 체계**: 조합 길이(연결 개수)에 따라 점수 차등(많을수록 높은 점수)을 줄지 결정할 것.

**예시 요청**: "사과 없앨 때 몇 점인지 플로팅돼서 보여줘", "플로팅 점수는 마지막으로 제거된 사과 위치에서 바로 떠올라야 돼. 더 강조해서 보여줘. 사과에 가리기도 하고", "합친 사과의 개수가 많을수록 더 높은 점수를 줄 수 있도록 수정해줘"

---

## 5. 문서화·재현성

- **완전 재현용 명세**: 다른 사람·AI가 똑같이 만들 수 있도록, 전체 소스 코드를 포함한 md 문서를 준비할 것.
- **게임 설명 포함**: 규칙, 점수, 레벨, UI/UX, 기술 스택 등을 같은 문서에 정리할 것.

**예시 요청**: "완전히 똑같이 만들 수 있게 파일 수정해줘", "plan.md처럼 게임에 대한 설명 부분도 추가해줘"

---

## 6. 기타

- **완료 알림**: `task-completion-notify` 규칙에 의해 모든 작업 완료 시 소리+팝업 알림이 자동 적용됨. 게임 작업 시에도 동일.
- **레벨·목표**: 목표 점수 간격(예: 20점 단위), 레벨별 그리드·숫자 범위를 명확히 할 것.

---

## 활용 방법

- 새 게임을 만들 때: 위 항목 중 해당하는 것만 골라 "이 체크리스트 기준으로 구현/수정해줘"라고 요청.
- 기존 게임을 다듬을 때: "플레이 가능성", "조작 UX", "시각 디자인", "피드백" 순으로 하나씩 점검해 달라고 요청.
