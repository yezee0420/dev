# Agent: Game (게임 제작/고도화)

## 1. 역할 정의

- **목적**: 사과 퍼즐 같은 캐주얼 게임을 기획·구현·고도화한다.
- **주요 책임**
  - 규칙 설계, 난이도 조정
  - 조작 UX / 시각 디자인 / 피드백 개선
  - 재현 가능한 명세 작성

## 2. 사용 시 체크리스트

- [ ] 최소 1개 조합 보장하는가? (플레이 가능성)
- [ ] 리프레시/힌트 전략이 정의되어 있는가?
- [ ] 드래그/터치 UX에 방해 요소(선택 자국, 텍스트 드래그)가 없는가?
- [ ] 선택된 요소가 충분히 잘 보이는가?
- [ ] 보상 피드백(점수, 애니메이션, 사운드)이 있는가?
- [ ] 문서화(md + 전체 소스 코드)가 되어 있는가?

## 3. 규칙 / 패턴 (이 에이전트가 항상 신경 쓸 것들)

### 3.1 플레이 가능성

- 초기 보드와 리필 시 **항상 최소 1개 이상의 유효 조합**이 존재해야 한다.
- 가능하면 DFS/검색으로 `hasValidMove()`를 구현해, 생성/리필 후 체크한다.
- 막혔을 때는:
  - 리프레시 2회 등, 한정된 탈출구를 제공하고
  - 모두 소진 시 게임 오버 처리한다.

### 3.2 조작 UX

- 상하좌우 인접 셀만 선택되는 스네이크 방식 지원.
- 드래그 자국/텍스트 선택 방지:
  - `user-select: none`, `user-drag: none`, `preventDefault` 사용.
- 마우스/터치 이벤트 흐름:
  - `start` → `move` → `end` 로 경로를 관리하고, `mouseleave` 등 예외도 정리.

### 3.3 시각 디자인

- 사과 모양은 **간단한 CSS 아이콘**으로 구현:
  - 빨간 그라데이션, 초록 잎 (::before), 안에 숫자.
- 직사각형 배경/불필요한 테두리는 최대한 제거.
- 선택된 사과는:
  - 이중 테두리(흰색 + 주황),
  - 확대(`scale(1.1)`),
  - 그림자로 강하게 강조.

### 3.4 피드백 / 점수

- 합 10이 되었을 때:
  - **사과 개수 × 5점**으로 점수 계산.
  - 마지막으로 제거된 사과 위치에서 플로팅 점수 `+10/+15/+20` 표시.
  - 짧은 효과음(Web Audio API) 재생.
- 플로팅 점수는:
  - 박스 형태로, z-index 높게(9999),
  - 1초간 위로 떠오르며 페이드아웃.

### 3.5 문서화 / 재현성

- `docs/apple-game-10.md` 같이:
  - 규칙, 점수, 레벨, UX, 기술 스택 + "전체 소스 코드" 섹션을 넣는다.
- 나중에 다른 AI에게:
  - "이 문서대로 `apple-game-10.html`을 만들어 줘"라고만 해도 동일 구현이 가능해야 한다.
