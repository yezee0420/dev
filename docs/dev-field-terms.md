## 🔄 4. 데이터 연동 (싱크 계열)
| 용어 | 뜻 | 예시 |
|---|---|---|
| **sync** | 여러 곳 값 동기화 | "탭끼리 값 싱크 맞춰줘" |
| **bind / binding** | 값을 묶어서 연동 | "input이랑 차트 바인딩" |
| **state** | 화면이 기억하는 값 | "이거 스테이트로 빼줘" |
| **shared state** | 여러 컴포넌트가 공유하는 값 | "전역 셰어드 스테이트로" |
| **single source of truth** | 값의 출처를 하나로 | "한 곳에서만 관리하게" |
| **two-way binding** | 양방향 연동 | "양방향 바인딩" |
| **derived / computed** | 다른 값에서 자동 계산 | "합계는 computed로" |
| **reactive** | 값 바뀌면 자동 갱신 | "리액티브하게 연동" |
| **debounce** | 연속 입력 마지막만 반영 | "검색에 디바운스" |
| **throttle** | 일정 간격으로만 실행 | "스크롤에 쓰로틀" |
| **optimistic update** | 서버 응답 전 미리 화면 갱신 | "옵티미스틱 업데이트" |
| **handler** | 이벤트/요청을 받아 처리하는 함수 | "제출 핸들러 만들어줘" |

> 💡 **"핸들러로 모든 데이터 연동"** 의 정확한 표현:
> → **"하나의 shared state로 묶고, 핸들러에서 업데이트하면 다 sync되게 해줘"**

## 🎯 실전 조합 예시
- *"리스트에 **스켈레톤** 깔고, 데이터 오면 **페이드인**, 빈 데이터면 **empty state** 보여줘"*
- *"슬라이더 움직이면 차트가 **리액티브하게 sync**되고, 목표는 **threshold line**으로"*
- *"저장 버튼 누르면 **옵티미스틱 업데이트** 하고 **토스트** 띄워줘"*
- *"카테고리는 **드롭다운**, 선택하면 **하이라이트**, 상세는 **드릴다운**"*
- *"입력값들 하나의 **shared state**로 묶고 **핸들러**에서 업데이트하면 전 탭 **sync**되게"*
