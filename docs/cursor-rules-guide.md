# Cursor 규칙에 넣을 수 있는 것들 — 완전 쉽게 설명

> **이 문서는 누구를 위한 건가요?**  
> Cursor를 10년 써 온 개발자, 코딩을 처음 하는 비개발자, 그리고 “규칙이 뭐예요?”라고 물을 만한 분까지, **다 같이** 이해할 수 있도록 썼어요.

---

## 🎯 먼저: "규칙"이 뭐예요? (유치원 수준)

**쉽게 말하면:**  
규칙은 **"이렇게 해 주세요"라고 AI한테 미리 적어 두는 메모**예요.

- **비유:** 선생님이 칠판에 "숙제는 연필로 쓰기"라고 적어 두면, 우리는 그걸 보고 따라 하는 것처럼요.
- **Cursor에서:** `.cursor/rules/` 폴더에 **규칙 파일**을 넣어 두면, AI가 그 프로젝트에서 일할 때 **그 메모를 읽고 그대로 따라요**.

**한 줄 요약:**  
규칙 = "이 프로젝트에서는 이렇게 해 달라"고 AI에게 주는 **영구 메모**.

---

## 📂 규칙은 어디에, 어떻게 넣나요?

- **위치:** 프로젝트 안의 **`.cursor/rules/`** 폴더
- **파일 이름:** 자유 (예: `한국어-기본.mdc`, `코드스타일.mdc`)
- **파일 끝:** 반드시 **`.mdc`** (마크다운 비슷한 형식)

**한 줄 요약:**  
`.cursor/rules/` 폴더에 `이름.mdc` 파일을 만들면 그게 규칙이에요.

### ⚠️ 중요: 워크스페이스 루트에만 둔다

Cursor는 **워크스페이스 루트**의 `.cursor/rules/`만 적용합니다. 하위 프로젝트 폴더(예: `bugo-alert/.cursor/rules/`)에 둔 mdc는 **적용되지 않습니다**.

| 구조 | 적용 여부 |
|------|-----------|
| `dev/.cursor/rules/*.mdc` | ✅ 적용 (워크스페이스가 `dev`일 때) |
| `dev/bugo-alert/.cursor/rules/*.mdc` | ❌ 적용 안 됨 |

**권장:** 여러 하위 프로젝트가 있으면 워크스페이스 루트에 `프로젝트명-역할.mdc` 형태로 통합한다.  
예: `bugo-backend-agent.mdc`, `bugo-parser-agent.mdc` — globs에 `bugo-alert/app/**` 등 경로 지정.

## 1️⃣ 수준 1 — "항상 이렇게 해 주세요" (기본·공통)

프로젝트 **어디서든**, **무슨 파일을 열어도** 적용되게 하고 싶을 때 쓰는 내용이에요.

### 1-1. 언어 (무슨 말로 대답할지)

| 넣을 수 있는 내용 | 쉽게 말하면 | 예시 문장 (규칙에 적을 만한 것) |
|------------------|-------------|----------------------------------|
| 기본 응답 언어 | "평소에는 이 언어로 말해 줘" | "특별한 요청이 없으면 항상 **한국어**로 답변한다." |
| 코드/주석 언어 | "코드 설명이나 주석은 이 언어로" | "주석과 문서는 **한국어**로 쓴다. 변수명·함수명은 영어를 쓴다." |
| 전문 용어 | "이 단어는 우리가 쓰는 뜻으로 써 줘" | "우리 팀은 '배포' 대신 '릴리스'를 쓴다." |

**한 줄 요약:**  
"한국어로 써 줘", "주석은 한국어", "이 단어는 이렇게 써 줘" 같은 **말/글 쓰기 방식**을 규칙에 넣을 수 있어요.

---

### 1-2. 답변 스타일 (얼마나 짧게/길게, 순서는 어떻게)

| 넣을 수 있는 내용 | 쉽게 말하면 | 예시 문장 |
|------------------|-------------|-----------|
| 요약 먼저 | "결론부터 한 줄로, 그다음 자세히" | "먼저 **한 줄 요약**을 쓰고, 필요하면 그다음에 자세히 설명한다." |
| 길이 | "짧게 해 줘" / "설명 충분히 해 줘" | "설명은 **3문장 이내**로 한다." 또는 "필요한 만큼 **자세히** 설명한다." |
| 목록 사용 | "목록으로 정리해 줘" | "두 개 이상일 때는 **글머리표(불릿)** 로 나열한다." |
| 코드 보여주기 | "예시 코드 꼭 포함해 줘" | "방법을 설명할 때는 **동작하는 예시 코드**를 최소 하나 포함한다." |

**한 줄 요약:**  
"요약 먼저", "짧게", "목록으로", "예시 코드 포함"처럼 **답을 쓰는 방식**을 규칙으로 정할 수 있어요.

---

### 1-3. 문서/글쓰기 형식 (제목, 굵게, 코드 표기)

| 넣을 수 있는 내용 | 쉽게 말하면 | 예시 문장 |
|------------------|-------------|-----------|
| 제목 단계 | "큰 제목, 작은 제목 몇 단계까지 쓸지" | "제목은 `##`, `###` 수준만 쓴다. `#` 하나짜리(가장 큰 제목)는 문서 제목에만 쓴다." |
| 강조 | "중요한 단어는 굵게" | "핵심 용어는 **굵게** 표시한다." |
| 코드/파일 이름 | "코드나 파일 이름은 어떻게 표시할지" | "파일명, 폴더명, 함수명은 **백틱(`)** 으로 감싼다. 예: `index.js`." |
| 표 | "표를 쓸지, 안 쓸지" | "비교할 항목이 2개 이상이면 **표**로 정리한다." |

**한 줄 요약:**  
제목 크기, 굵게, 백틱, 표 사용 같은 **글꼴/형식**을 규칙에 넣을 수 있어요.

## 2️⃣ 수준 2 — "코드를 다룰 때 이렇게 해 주세요" (코드 스타일)

**특정 종류의 파일**(예: `.ts`, `.tsx`, `*.html`)을 열었을 때만 적용되게 할 수도 있고, "항상 적용"으로 둘 수도 있어요.

### 2-1. 이름 짓기 (변수, 함수, 파일)

| 넣을 수 있는 내용 | 쉽게 말하면 | 예시 문장 |
|------------------|-------------|-----------|
| 변수/함수 이름 스타일 | "낙타표기법 쓸지, 스네이크 쓸지" | "변수·함수는 **camelCase**(낙타표기법). 상수는 **UPPER_SNAKE_CASE**." |
| 파일/폴더 이름 | "파일은 어떻게 이름 지을지" | "파일명은 **kebab-case**(소문자-하이픈). 예: `user-profile.ts`." |
| 불리언 변수 | "참/거짓 변수는 is, has로 시작" | "참/거짓 값 변수는 **is~**, **has~** 로 시작한다. 예: `isLoading`, `hasError`." |

**한 줄 요약:**  
변수명, 함수명, 파일명을 **어떤 표기법으로** 쓸지 규칙으로 정할 수 있어요.

---

### 2-2. 코드 구조 (들여쓰기, 줄 길이, 빈 줄)

| 들여쓰기 | "스페이스 2칸 vs 4칸, 탭 쓰기" | "들여쓰기는 **스페이스 2칸**을 쓴다. 탭은 쓰지 않는다." |
| 한 줄 길이 | "한 줄이 너무 길어지지 않게" | "한 줄은 **80자 또는 120자**를 넘기지 않는다." |
| 빈 줄 | "함수 사이, 논리 블록 사이에 빈 줄" | "함수와 함수 사이에는 **빈 줄 하나**를 둔다." |

**한 줄 요약:**  
들여쓰기, 한 줄 길이, 빈 줄 같은 **코드 모양**을 규칙에 넣을 수 있어요.

---

### 2-3. 주석 (무엇을, 어떻게 쓸지)

| 넣을 수 있는 내용 | 쉽게 말하면 | 예시 문장 |

| 복잡한 부분 | "어려운 로직은 설명 필수" | "복잡한 계산·조건은 **한 줄 요약 주석**을 달아 준다." |

### 2-4. 에러·예외 처리 (실패했을 때 어떻게 할지)

## 5️⃣ 규칙 파일 설정 요약 (프론트매터)

각 규칙 **파일 맨 위**에 넣는 설정이에요. "언제 이 규칙을 적용할지"를 정합니다.

| 설정 | 쉽게 말하면 | 예시 |
|------|-------------|------|
| `alwaysApply: true` | "**항상** 이 규칙 적용해 줘" | 언어, 답변 스타일 같은 공통 규칙 |
| `alwaysApply: false` + `globs` | "**이런 파일** 열렸을 때만 적용해 줘" | `globs: **/*.ts` → TypeScript 파일일 때만 |
| `description` | "이 규칙이 뭔지 한 줄로" | 규칙 목록에서 보이는 설명 글 |

**globs 예시:**

- `**/*.ts` → 모든 `.ts` 파일
- `**/*.tsx` → 모든 `.tsx` 파일
- `src/components/**/*` → `src/components/` 아래 모든 파일
- `**/*.html` → 모든 HTML 파일

**한 줄 요약:**  
`alwaysApply` = "항상 할지", `globs` = "어떤 파일에서 할지"를 정하는 거예요.

## 6️⃣ 정리: 규칙에 넣을 수 있는 것 체크리스트

아래를 보면서 "우리 프로젝트에는 이걸 넣고 싶다" 하는 것만 골라서 규칙 파일에 적으면 됩니다.

- [ ] **언어:** 기본 응답 언어(한국어/영어 등), 주석·문서 언어
- [ ] **답변 스타일:** 요약 먼저, 길이, 목록, 예시 코드 포함 여부
- [ ] **문서 형식:** 제목 단계, 굵게, 백틱, 표
- [ ] **이름:** 변수/함수/파일 표기법(camelCase, kebab-case 등)
- [ ] **코드 구조:** 들여쓰기, 줄 길이, 빈 줄
- [ ] **주석:** 언어, 무엇을 쓸지(이유 vs 동작)
- [ ] **에러 처리:** 로그, 사용자 메시지
- [ ] **테스트:** 필수 여부, 이름 규칙, 언어
- [ ] **API:** URL 스타일, HTTP 메서드, 응답 형식
- [ ] **보안:** 비밀값, 입력 검증
- [ ] **접근성·UI:** alt, 버튼/링크 문구, 키보드
- [ ] **성능:** 이미지, 큰 목록
- [ ] **프로젝트 구조:** 폴더 역할, 파일 나누기
- [ ] **팀 용어:** 통일할 단어, 쓰지 말 말
- [ ] **도구·버전:** 쓸 라이브러리, 기준 버전
