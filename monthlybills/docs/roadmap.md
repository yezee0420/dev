# Roadmap

> 개발자 1명, 풀타임 아님 기준. 기간은 가이드라인이며 실측으로 보정.

## Phase 0 — 기획 (현재, ~1주)

- [x] 프로젝트 폴더·README
- [ ] 로드맵·BM·스키마·온보딩 플로우 문서화
- [ ] 구독 시드 카탈로그 (상위 30–40개)
- [ ] 레퍼런스 UI 스케치 (모바일 우선)
- [ ] Supabase 프로젝트 생성, Kakao/Naver 개발자 앱 신청

**산출물**: `docs/` + `data/seed-subscriptions.json`

## Phase 1 — Web MVP + 소셜 로그인 (~3–4주)

**목표**: 가입 후 60초 안에 "내 월 고정비 합계"를 보여준다.

- [ ] Next.js 14 부트스트랩 (`app/`)
- [ ] Supabase 연동 (Auth + DB)
- [ ] **Kakao 로그인** (Supabase 네이티브 OAuth)
- [ ] **Naver 로그인** (커스텀 OAuth provider)
- [ ] 온보딩 플로우 3 스텝:
  1. 구독 선택 (카탈로그에서 복수 선택)
  2. 직접 추가 (카탈로그 외)
  3. 고정비 입력 (월세/관리비/통신비/인터넷/카드 연회비 등) — skip 가능
- [ ] 상단 월/연 총합 카드
- [ ] 구독·고정비 CRUD
- [ ] 기본 리스트 뷰 + 카테고리 필터

**Done 기준**: 본인 포함 지인 5명이 실제로 가입해 사용하고 있음.

## Phase 2 — 분석·캘린더 (~2주)

- [ ] 결제일 캘린더 뷰
- [ ] 카테고리별 지출 도넛 차트
- [ ] 월/연 트렌드 그래프
- [ ] 연간 총 지출 리포트 (연말/연초 공유용 이미지 내보내기)
- [ ] **브랜드명 확정** (이 단계에서 결정 — 마케팅 시작 직전)

## Phase 3 — 알림 (~1–2주)

**무료 발사대**: 웹푸시 + 이메일부터. 사용자 수 확인 후 카카오 알림톡 유료 전환.

- [ ] 웹푸시 (VAPID, 무료)
- [ ] 이메일 알림 (Supabase Functions + Resend 무료 티어)
- [ ] (유저 수 200+ 시점에) 카카오 알림톡 — 솔라피/알리고 경유 (건당 8–15원)
- [ ] 알림 설정 페이지: D-3 / D-1 / D-day 선택

## Phase 4 — Switch & Save + 수익화 (~2주)

- [ ] 더 싼 요금제 추천 규칙 엔진
  - 예: 유튜브 프리미엄 → 가족 요금제 전환 가이드
  - 예: OTT 중복 감지 → 해지 추천
- [ ] 제휴 링크 클릭·전환 트래킹
- [ ] 프리미엄 티어 결제 연동 (토스페이먼츠)

**수익 지표 첫 측정**: ARPPU, 제휴 전환율.

## Phase 5 — 앱 (~3–4주)

- [ ] Expo 프로젝트 부트스트랩
- [ ] 웹 컴포넌트 → React Native 포팅 (Tailwind → NativeWind)
- [ ] 푸시 알림 (Expo Push)
- [ ] App Store / Play Store 등록

## Phase 6+ — 장기 (일정 미정)

- 오픈뱅킹 연동: 카드 내역에서 구독 자동 감지 (별도 프로젝트급, 심사·보안 부담)
- 가족 공유 플랜
- 카드사·뱅크샐러드 white-label 제휴 탐색

## 의사결정이 필요한 체크포인트

| 시점 | 결정 사항 |
|---|---|
| Phase 1 끝 | 지인 피드백 기반 온보딩 단순화 여부 |
| Phase 2 끝 | 브랜드명·로고·도메인 |
| Phase 3 중 | 유료 알림톡 전환 시점 (MAU·리텐션 기준) |
| Phase 4 끝 | PMF 판정. 안 나면 B2B2C 피벗 검토 |
