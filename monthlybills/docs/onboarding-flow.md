# Onboarding Flow

> 목표: **60초 안에 사용자가 "내 월 고정비 합계"를 본다.** 긴 설문 X, 복수 선택과 스킵을 적극 활용.

## 전체 화면 구성

```
[0] 랜딩 → [1] 로그인 → [2] 구독·고정비 입력 → [3] 직접 추가 → [5] 대시보드
```

3단계는 언제든 스킵 가능. 모두 스킵해도 대시보드는 노출 (단, 총합 0원).

> **구현 메모 (2026-04-21)**: Step 2와 Step 4(고정비 입력)를 한 화면으로 병합함. 구독 카테고리 아코디언 아래에 고정비 빠른 입력 칩을 함께 배치 → 플로우 1단계 감소. `/onboarding/fixed-costs`는 별도 라우트로 만들지 않음.

---

## [0] 랜딩 페이지

- 한 줄 후킹: "내 구독과 고정비, 한 화면에."
- CTA: **"내 월 고정비 계산하기"** → 로그인으로 이동
- 비로그인 데모: 실제 로그인 없이 [2]→[5]까지 둘러볼 수 있는 "체험" 링크 (로컬스토리지 저장, 나중에 로그인 시 이전 가능)

## [1] 로그인

- **카카오로 시작하기** (primary, 노란색)
- **네이버로 시작하기**
- (후순위) 이메일 로그인

전환율 최적화: 카카오/네이버 버튼만 노출. 이메일은 "다른 방법" 뒤에 숨김.

## [2] 구독·고정비 입력 — 레퍼런스 이미지 기반

상단 탭: **둘러보기** / **내 항목 (n)** — n은 구독 + 고정비 합계

- 검색 바: "서비스명으로 검색"
- **인기 서비스** (가로 스크롤 칩): 넷플릭스 · 유튜브 프리미엄 · 쿠팡 로켓와우 · 디즈니+ · 티빙 · 웨이브
- **카테고리 아코디언**:
  - 🛵 배달/쇼핑 (쿠팡, 마켓컬리, 배민클럽 등)
  - 🎬 OTT (넷플릭스, 디즈니+, 웨이브, 티빙, 왓챠 등)
  - 🎵 음악 (유튜브 뮤직, 멜론, 스포티파이, 애플뮤직 등)
  - 📚 독서 (밀리의서재, 리디셀렉트, 예스24 크레마클럽)
  - 🎓 교육/생산성 (Notion Plus, ChatGPT Plus, Claude Pro 등)
  - ☁️ 클라우드 (iCloud, 구글 원, 드롭박스, OneDrive)

**인터랙션**:
1. 카테고리 탭 → 펼쳐지며 서비스 리스트 (카드 형태)
2. 카드 탭 → 요금제 선택 모달 (`plans` jsonb 기반, 예: "베이직 9,500원 / 스탠다드 13,500원")
3. 선택되면 상단 **"내 구독 (n)"** 카운트 증가

### 고정비 섹션 (같은 화면 하단)

- 상단 카피: "그 외 매달 나가는 돈, 같이 넣어두면 총액이 더 정확해져요."
- 프리셋 칩: 🏠 월세 · 🏢 관리비 · 💳 대출 이자 · 📞 통신비 · 🌐 인터넷 · 💳 카드 연회비
- 칩 탭 → 인라인 폼 오픈 (금액 + 결제 주기 월/연 토글)
- **➕ 기타**: 커스텀 이름 입력 후 추가, 동일한 인라인 폼

하단 고정 버튼: **"다음"** (0개여도 진행 가능, "선택한 항목이 없어요. 그래도 진행할까요?" 컨펌 1회)

## [3] 직접 추가

카탈로그에 없는 구독 자유 입력.

- 상단: **"빠진 구독이 있나요? 직접 추가해보세요."**
- 입력 폼:
  - 서비스명 (필수, text)
  - 카테고리 드롭다운
  - 금액 (필수, 숫자)
  - 결제 주기 (월/연)
  - 결제일 (선택)
- [+ 추가] 버튼으로 여러 개 쌓기 가능, 리스트로 아래에 누적 표시
- 하단: **"건너뛰기"** / **"다음"**

## [4] 고정비 입력 — Step 2에 통합됨

구현은 Step 2의 "고정비 섹션"을 참고. 별도 라우트로 만들지 않음.

## [5] 대시보드

- 상단 **총합 카드 (hero)**:
  - **월 합계** ₩ XXX,XXX (크게)
  - **연 합계** ₩ X,XXX,XXX (sub)
  - 카테고리별 작은 바 차트 (Phase 1은 막대 / Phase 2에서 도넛)
- **구독 리스트** (카테고리별 섹션)
- **고정비 리스트**
- FAB **[+ 추가]**: 구독 또는 고정비 추가
- 설정 아이콘: 알림 설정(Phase 3 전까지는 "준비 중"), 로그아웃

## 상태 / 라우팅 규칙

- `profiles.onboarding_completed = false` → 랜딩 접근 시 자동으로 [2]로 리디렉트
- 각 단계 뒤로가기 가능, 입력한 값은 클라이언트 상태(+ 주기적 draft 저장)로 유지
- [2]/[3]/[4] 모두 빈 상태로 [5] 도달 시: 빈 대시보드 + "첫 항목을 추가해보세요" 안내

## 카피 톤

- 친근한 반말은 피하기 (돈 얘기라서 신뢰감 우선)
- "해요", "했어요" 체 통일
- 금액은 언제나 `₩` + 천단위 콤마

## Phase 1에서 미구현 (Phase 2로 이월)

- 결제일 캘린더 뷰
- 알림 설정 플로우
- 직접 입력 시 카탈로그 "이 서비스일까요?" 자동 매칭 제안
