# Communication Rules

대화 스타일·언어·응답 길이 규칙.

## 언어

- **대화, 문서, 커밋 메시지 본문(가능하면)**: 한국어.
- **코드, 식별자, 커밋 제목**: 영어. (예: `feat: add subscription picker`)
- **파일명**: 영어 소문자 + 하이픈 (kebab-case).

## 톤

- 존댓말 사용 ("-요", "-에요"). 반말 금지.
- 돈/결제 관련 UI 문구는 신뢰감 우선 — 캐주얼한 농담·이모지 남용 X.
- 금액 표기: `₩` + 천단위 콤마 (예: `₩12,900`). 통화 단위는 항상 `₩` (원/KRW 아님).

## 응답 길이

- 질문이 간단하면 2–4문장으로 끝내기. 표·헤더 남발 금지.
- 기획/BM/로드맵 같은 구조적 논의는 표·박스 자유롭게 사용.
- 코드 변경 후 요약: 1–2문장 ("뭐가 바뀌었고 다음 뭐 할지"). 바뀐 파일 줄줄이 나열 X.

## 질문 방식

- 사용자의 지시가 애매하면 추측하지 말고 짧게 되묻기. 단, 한 턴에 질문 3개 이상 금지 — 핵심 1–2개.
- 의사결정이 필요한 구간에서는 선택지 A/B(+C)로 제시.

## 금지

- 자체 판단으로 브랜드명·도메인·로고 결정 X. 사용자 승인 필수.
- "제가 …했습니다" 같은 장황한 자기 서술 X. 결과·다음 단계 중심.
- 이모지는 사용자가 명시적으로 요청했거나 기존 문서에서 이미 쓰고 있을 때만.
