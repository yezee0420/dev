# UX Writing Rules

사용자에게 노출되는 화면 텍스트 규칙.

## 톤

- 존댓말 ("-요", "-어요"). 반말 금지.
- 친근하되 신뢰감 우선 — 돈 관리 앱이라는 특성 반영.
- 이모지는 카테고리 아이콘 수준으로만. 카피 본문에 남용 금지.

## 금액 표기

- 통화 기호: 항상 `₩` (앞에 붙임, 공백 없음) — 예: `₩12,900`
- 천단위 콤마 필수.
- 월/연 환산 표기: "/월", "/년" 접미 (예: `₩17,000/월`, `₩120,000/년`).
- 0원은 "무료"로 표기 (예: 올리브영 멤버십).

## 주기 표기

- `monthly` → "매달", "월 단위"
- `yearly` → "매년", "연 단위"
- `quarterly` → "분기마다"
- 결제일: "매달 15일 결제" 형태.

## 금지·허용 문구

- ❌ "결제하세요" (명령형 압박) → ✅ "결제일이 다가와요"
- ❌ "해지해드렸습니다" (앱이 대신 해지한 것처럼) → ✅ "해지 완료 표시했어요"
- ❌ "무료로 평생 써보세요" (과장) → ✅ "기본 기능은 무료로 계속 쓸 수 있어요"
- ❌ "놓치지 마세요!!!" → ✅ "결제일 3일 전에 알려드려요"

## 브랜드명 표기

- 브랜드 확정 전까지 UI에는 **"monthlybills (가제)"** 또는 공백 사용.
- 공식 브랜드명 등장 시 이 규칙을 갱신.

## 서비스 이름 표기 (catalog)

- 공식 한국어 명칭 우선 (예: "넷플릭스", "유튜브 프리미엄", "쿠팡 로켓와우").
- 영문 병기가 필요하면 괄호 (예: `iCloud+ (아이클라우드 플러스)`).
- 대소문자는 공식 표기 따르기 (Apple TV+, ChatGPT, GitHub).

## 에러·빈 상태

- 에러: 원인 + 사용자가 할 수 있는 다음 액션. 영어 스택트레이스 노출 금지.
- 빈 상태: "아직 추가한 구독이 없어요. 지금 추가해볼까요?" 식으로 CTA 포함.
